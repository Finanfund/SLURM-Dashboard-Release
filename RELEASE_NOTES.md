# SLURM Dashboard Release Notes

## v18 — SSH 稳定性修复 + 作业数据防丢失

### 问题修复
- **修复 SSH 通道饱和导致的作业 CPU/内存数据为 0 的问题**
  - 当 `refreshIntervalSec` 设为 1s 时，多个并发采集周期在同一 SSH 连接上打开过多通道，导致所有节点 SSH 命令超时
  - 新提交的作业无法获取首次 cgroup 数据，cpu_percent 和 mem_used_gb 永远显示 0
  
- **SSHPool 节点级并发限制**（方案 D）：每个节点最多 2 个并发 SSH 命令，防止通道饱和
  
- **SSH 退化检测与自动降频**（方案 B）：连续 5 个采集周期全部缓存时，自动将实际采集间隔提升到至少 5 秒，避免恶性循环；SSH 恢复后自动回落
  
- **兜底路径修复内存数据**（方案 E）：最终兜底路径现在同时修复 `cpu_percent` 和 `mem_used_gb`（之前只修复 CPU）
  
- **冷启动内存推算**（方案 C.2）：当新 job 被分配到缓存节点且该节点上仅有此一个 job 时，使用节点级内存近似为该 job 的 mem_used_gb

### 配置变更
- `MIN_REFRESH_INTERVAL` 从 1 调整为 2 秒（1s 在多节点场景下必然引起 SSH 通道饱和）

## v17 — 低功耗模式 + 性能优化

### 新特性
- **低功耗模式**：导航栏新增闪电图标切换按钮，一键切换高性能/低功耗模式
  - 移除所有 `backdrop-filter: blur()` 毛玻璃效果（最大的 GPU 开销来源）
  - 禁用所有非必要动画和过渡效果
  - 简化 box-shadow、text-shadow 等发光效果
  - 替换半透明背景为实色背景（GitHub 暗色主题风格）
  - 简化渐变进度条/节点柱状条为纯色
  - 移除 `background-attachment: fixed` 避免滚动重绘
  - 状态通过 `localStorage` 持久化记忆

### 性能优化（高性能模式也生效）
- CSS `transition: all` 替换为具体属性列表（避免浏览器检查所有属性）
- 节点卡片添加 `will-change: transform` 提示 GPU 合成
- 表单控件、按钮、卡片的 transition 均指定精确属性

## v16.1 — 粘性工具栏布局 + UI 修复

### 新特性
- **粘性工具栏布局**：所有 4 个 tab 页（集群状态、任务列表、历史任务、文件浏览）采用固定工具栏 + 独立滚动内容区设计
  - 工具栏/搜索栏/过滤按钮固定在顶部，不随内容滚动
  - 内容区域独立纵向滚动，无高度限制
  - 表格列表的表头（thead）在滚动时粘顶显示

### Bug 修复
- **NUMA 堆叠图本地内存占满跳变**：修复了 WS 实时追加数据点缺少 NUMA 字段时，内存图表本地部分瞬间占满的问题。改用最近已知的 local/remote 比例拆分
- **任务颜色交换**：修复了节点图表中堆叠面积图 series 顺序不稳定导致作业颜色互换的问题。渲染前按 `jid` 数字序排序
- **锁定按钮记忆**：日志锁定/跟随按钮现在跨任务详情模态框保持状态，不再每次打开都重置
- **已结束任务自动全程历史**：打开已结束任务详情时默认加载全程历史数据

## v16.0 — 初始发布

### 核心功能
- **集群状态监控**：实时 CPU/内存使用率、节点状态、分区概览
- **任务管理**：作业列表、详情查看、日志查看、取消作业
- **文件浏览器**：远程文件浏览、编辑、上传、下载、收藏夹
- **历史任务**：任务归档记录、搜索、详情回溯
- **NUMA 分析**：按需 NUMA 内存分布分析 + 可选趋势记录
- **实时日志**：自动采集并刷新 stdout/stderr 输出
- **智能缓冲提示**：自动检测 PYTHONUNBUFFERED 环境变量
- **paramiko SSHPool**：持久 SSH 连接池，采集延迟 ~1.5s
- **5 层 0% CPU 保护**：防止采集异常导致的虚假 0% 数据
- **ECharts 图表**：增量 merge 模式无闪烁更新
- **Golden Angle 颜色系统**：137.508° 间隔确保相邻作业颜色差异最大化
